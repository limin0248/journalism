GitHub Pages 部署流程详解

以部署 journalism 仓库（finance.sensible.cyou）为例，完整梳理每一步的操作含义。

◈ 共 10 个步骤 ◈ 预计耗时 10-15 分钟 ◈ 2026年7月
前置条件：本地项目已初始化 Git 仓库 · 代码已推送到 GitHub · 已购买并持有域名
1

创建 GitHub 仓库

操作路径

GitHub 右上角 + 下拉菜单 → New repository

填写内容

字段含义建议值
Repository name仓库名称（唯一标识）journalism
Description项目描述（可选）财经记者炼成记
Public / Private公开/私有Public（GitHub Pages 免费版需要公开）
Add a README初始化 README不要勾选（本地已有则冲突）
Add .gitignore忽略文件模板不要勾选
Choose a license开源协议不要勾选

点击按钮

Create repository（绿色按钮）

创建完成后，GitHub 会显示一个 HTTPS 地址，如 https://github.com/limin0248/journalism.git
2

推送本地代码到远程

# 添加远程仓库地址 git remote add origin https://github.com/limin0248/journalism.git # 将本地分支重命名为 main（GitHub 默认） git branch -M main # 推送并建立关联 git push -u origin main

命令含义

命令含义
git remote add告诉本地 Git：远程仓库在 GitHub 的哪个地址
git branch -M main把本地分支名改成 main（GitHub 默认分支名）
git push -u origin main把本地代码推送到远程的 main 分支，并建立追踪关系
3

进入 GitHub Pages 设置

操作路径

仓库页面 → Settings（顶部标签栏） → 左侧侧边栏 → Pages

路径含义

页面功能
Settings仓库的完整配置中心
PagesGitHub 的静态网站托管服务配置入口
4

配置发布源（Source）

界面内容

Source 下拉框 → 选择 Deploy from a branch

含义

GitHub 提供两种部署方式：

方式说明适用场景
Deploy from a branch从某个分支的文件夹自动部署静态网站（HTML/CSS/JS）
GitHub Actions通过 CI/CD 流程自定义构建需要编译的项目（如 Vue、React、Jekyll）
选择原因：本项目是纯静态 HTML 网站，选择 Deploy from a branch 最简单，GitHub 会自动读取分支中的文件并部署。
5

选择分支和文件夹

界面内容

下拉框选项含义
Branchmain使用哪个分支的代码
Folder/(root)使用分支根目录作为网站根目录

含义

Branch：GitHub 默认用 main 作为主分支，代码推送到这里

/(root)：表示网站根目录就是 main 分支的根目录（index.html 直接放在仓库根目录下）

点击按钮

Save（灰色按钮）

保存后，GitHub 开始构建和部署。页面会刷新，显示部署状态。
6

配置自定义域名（Custom domain）

操作路径

Pages 设置页面 → 向下滚动到 Custom domain 区域

界面内容

输入框 → 填写：finance.sensible.cyou

点击按钮

Save（灰色按钮）

含义

概念说明
默认地址不配置域名时，网站地址是 https://limin0248.github.io/journalism
自定义域名绑定自己的域名，让用户通过 finance.sensible.cyou 访问
CNAME 文件GitHub 会自动在仓库中创建/更新 CNAME 文件，内容就是域名
此时 GitHub 会尝试检测域名的 DNS 解析是否指向 GitHub，如果 DNS 还没配置，会显示 DNS check unsuccessful
7

配置 DNS 解析（域名服务商处）

操作路径

阿里云域名控制台 → 找到 sensible.cyou → 解析 → 添加记录

填写内容

字段值含义
记录类型CNAME将域名指向另一个域名（而不是 IP 地址）
主机记录finance子域名前缀，finance.sensible.cyou
记录值limin0248.github.io指向 GitHub Pages 的服务器地址
TTL默认（10分钟）DNS 缓存时间，越小生效越快

点击按钮

确认 / 保存

含义

CNAME 记录：告诉 DNS 服务器，finance.sensible.cyou 实际上对应 limin0248.github.io

用户访问 finance.sensible.cyou 时，DNS 会将其解析到 GitHub 的服务器

8

验证 DNS 配置

操作路径

回到 GitHub Pages 设置页面 → 黄色横幅 → Check again

可能的状态

状态含义处理方式
DNS check unsuccessfulDNS 还没生效等待几分钟，再点击 Check again
DNS check successfulDNS 解析正确验证通过，可以下一步

原因

DNS 解析从配置到全球生效需要时间（通常几分钟到10分钟）。GitHub 需要确认你的域名确实指向了 GitHub 的服务器，才能继续绑定。

9

启用 HTTPS（Enforce HTTPS）

操作路径

DNS 验证通过后 → 勾选 Enforce HTTPS 复选框

含义

选项说明
不勾选网站可以通过 http:// 和 https:// 两种协议访问
勾选后强制所有访问走 https://，自动加密，更安全

前提条件

必须等 DNS check successful 后才能勾选，否则复选框是灰色不可用状态。

10

等待部署完成并访问

部署状态检查

GitHub Pages 设置页面顶部会显示：

Your site was last deployed to the github-pages environment by the pages build and deployment workflow.

访问网站

在浏览器中访问：

https://finance.sensible.cyou

部署时间

阶段耗时
代码推送到 GitHub 后约 1-2 分钟 自动构建
首次绑定自定义域名约 2-5 分钟 完成 SSL 证书申请和部署

后续更新代码

代码修改后，只需执行：

git add . git commit -m "描述修改内容" git push

GitHub Pages 会自动重新部署，约 1-2 分钟 后生效。

?

常见问题

DNS check 一直失败
DNS 解析未生效。等待 5-10 分钟后再次点击 Check again。
网站显示 404
没有 index.html 或文件路径错误。确保仓库根目录有 index.html。
样式/图片不加载
路径错误（用了绝对路径）。检查 CSS/JS 路径是否为相对路径 ./css/style.css。
旧内容还在
浏览器缓存。按 Ctrl+Shift+R 强制刷新。
HTTPS 无法勾选
DNS 还没验证通过。等 DNS check successful 后再勾选。
